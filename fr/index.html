Ne demande pas pour demander, juste demande.

De temps en temps, dans des canaux que je fréquente, je vois quelqu'un apparaître et demander,

Foobar123 :

Aucun expert Java dans le coin ?

C'est une mauvaise formulation pour plusieurs raisons. Ce que la personne est vraiment en train de demander est,

Foobar123 :

Aucun expert Java qui serait prêt à examiner mon problème, quel qu'il soit, même s'il n'est pas relié à Java ou même si une personne ne connaissant rien à Java pourrait répondre à ma question?

Il y a une multitude de raisons pour lesquelles les personnes AYANT la réponse ne répondraient pas. En demandant ça, vous demandez plus que ce que vous ne pensez demander.

Vous demandez aux gens de prendre des responsabilités. Vous questionnez les gens sur la confiance qu'ils portent sur leurs compétences. Vous écartez aussi inutilement les autres. Je réponds souvent aux questions reliées à des langages ou des librairies que je n'ai jamais utilisés, car la réponse relève (pour le développeur) du bon sens.

Vous pouvez aussi être vu comme quelqu'un...

Foobar123 :

J'ai une question à propos de Java mais je suis trop fatigué pour formuler ma question avec des mots jusqu'à ce que quelqu'un soit présent dans le channel et qu'il soit capable de répondre

...qui est juste paresseux. Si vous n'êtes pas prêt à faire le travail nécessaire pour résoudre votre problème, pourquoi les autres devraient le faire ?

La solution n'est pas de demander pour demander, mais simplement de demander. Quelqu'un qui ne regarde que de temps en temps ce qui se passe sur le channel a peu de chance de répondre à votre question, tandis que décrire le problème peut éveiller son intérêt et l'amener à y répondre.

Donc, pour résumer, ne demandez pas "Aucun expert Java dans le coin ?", mais plutôt "Que devrais-je faire pour résoudre [problème] en Java ainsi que [autres infos importantes] ?"

D'autres problèmes similaires : Le problème XY, Pas de bonjour. Pour aller plus loin dans ce thème : Comment poser une bonne question ?, ou si vous avez plus de temps : Comment poser des questions intelligentes.
