不要为了提问而提问

在我窥屏的一些聊天群组中，时不时会有人提出类似于这样的问题：

Foobar123:

这里有没有 Java 高手？

无论你怎么解释，这真的不是个好问题。这个人实际上要问的是：

Foobar123:

有没有愿意帮我解决问题的 Java 高手？虽然我也不知道问题出在哪里，也许可能不是 Java 的问题。或者有没有不精通 Java 但愿意帮我解决一下这个问题的群友？

那些确实精通这方面的人会有一万种理由不接受你的问题。对于这个问题，你对他们要求的比你预想中的要多很多。

这意味着你要让回答你问题的人承担一定的责任。这个问题像是在挑战他们对这方面知识和能力的信心，同时也无意中将他们拒之门外。我经常回答一些和我从未使用过的语言或程序库有关的问题，因为那些问题的答案（对于我们程序员来说）往往都是常识。

这种问题还有一种形式：

Foobar123:

我有一个关于 Java 的问题，但我太懒了，不想用语言解释出来，除非频道 / 群组里有人可以回答我。

....天哪，这也太懒了吧。如果连你自己都不愿意帮助自己解决问题，那更何况别人呢？

这种问题是很好避免的。不要为了提问而提问，直接表达你的问题即可。那些在频道 / 群组里窥屏的人只是时不时来看看这里有什么事情发生，并不太可能会去回答那些 “问里有问” 的问题。往往那些带有实际描述的问题才会激起他们的兴趣，并让他们回答或协助解决。

因此，总的来说，不要问类似于 「有没有 Java 高手？」 的问题，而是去问 「如何使用 Java（以及其他相关的描述）去处理某个问题？」 的问题。

其他类似的问题：X-Y 问题（英文 中文）、提问切勿打招呼（英文）。
延伸阅读：Stack Overflow 如何问一个好问题？（英文）。
