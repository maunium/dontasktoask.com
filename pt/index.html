Não pergunte para pedir, apenas peça

De vez em quando, nas salas de bate-papo online em que eu ando, alguém aparece e diz algo assim

Foobar123:

Algum especialista em Java por aí?

Esta é uma má forma, por várias razões. O que a pessoa está realmente perguntando é,

Foobar123:

Quaisquer especialistas em Java que estejam dispostos a se comprometer a investigar         problema, seja o que for, mesmo que não seja realmente         relacionado a Java ou se alguém que não sabe nada sobre Java         poderia realmente responder a minha pergunta?

Existem muitas razões pelas quais as pessoas que TÊM conhecimento não admitir isso. Ao perguntar, você está pedindo mais do que pensa você está perguntando.

Você está pedindo às pessoas que assumam a responsabilidade. Você está questionando as pessoas confiança em suas habilidades. Você também está murando desnecessariamente pessoas fora. Costumo responder perguntas relacionadas a idiomas ou bibliotecas que nunca foram usadas, porque as respostas são (de um modo programador) message

Eu tenho uma pergunta sobre Java, mas estou com preguiça de realmente formalizá-lo em palavras, a menos que haja alguém no canal que possa responda

..que é apenas preguiçoso. Se você não estiver disposto a fazer o trabalho para resolver seu problema, problema, por que deveríamos?

A solução não é pedir, mas apenas perguntar. Alguém que está ocioso no canal e só de vez em quando olha o que está acontecendo é é improvável que você responda à sua pergunta "pedir para perguntar", mas a descrição do problema pode despertar seu interesse e levá-los a responder.

Então, para resumir, não pergunte "Quaisquer especialistas em Java ao redor?", mas pergunte "Como faço para [problema] com Java e [outras informações relevantes]?"

Outros problemas semelhantes: O problema XY, Ola não. Leitura adicional: Como faço uma boa pergunta?, ou se você tiver mais tempo: Como fazer perguntas da maneira inteligente.
