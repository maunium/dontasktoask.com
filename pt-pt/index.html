Não perguntes para perguntar, Só pergunta

De vez em quando, nos chats de bate-papo, aparece-me alguem a dizer:

Foobar123:

Alguem sabe java?

Na verdade o que ela quer dizer é na verdade,

Foobar123:

Quaisquer especialistas em Java que estejam dispostos a se comprometer em pesquisar o meu problema, seja lá o que for, mesmo que não esteja relacionado a Java ou se alguém que não sabe nada sobre Java poderia realmente responder à minha pergunta?

Existem muitas razões pelas quais as pessoas que TÊM o conhecimento não admitem isso. Ao perguntar, Tu estás a pedir mais do que pensas que estás a pedir.

Tu ao perguntar isto, estás a perguntar às pessoas para assumir a reponsabilidade. Estás a duvidar da confiabilidade das habilidades das pessoas. Estás tambem a remover outras pessoas sem necessidade. Eu recebo várias perguntas relacionadas com linguagens ou bibliotecas que nunca usei, porque as respostas tem haver com senso comum (de um programador)

Isto tambem pode ser interpretado como:

Foobar123:

Eu tenho uma pergunta sobre java mas eu sou muito preguiçoso para formalizar palavras, ao menos que tenha alguem no canal que seja capaz de responde-la

..o que é apenas preguiçoso. Se tu não queres dar o trabalho de fazer a pergunta, porque deveriamos responder?

A solução é não perguntar para perguntar, mas sim, perguntar diretamente. Alguem que não é ativo, e raramente vê o que se passa no canal, dificilmente vai responder á teu pedido para perguntar, mas se o problema já estiver descrito, ele vai ter mais vontade de responder

Então, para resumir, não perguntes "Alguem sabe java?", pergunta "Como eu resolvo tal problema com java [informação extra]?"

Problemas parecidos: O problema do XY, Não digas Olá. Saber mais: Como eu faço uma boa pergunta?, Ou se tiveres mais tempo: Como fazer perguntas de maneira inteligente.
