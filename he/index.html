אל תשאל בשביל לשאול. פשוט תשאל!

מדי פעם, בקבוצות וצ'אטים מקוונים שאני נמצא בהם, מישהו קופץ ואומר משהו כמו,

משתמש123:

יש מומחי Java בסביבה?

זוהי צורה שגויה, מכמה סיבות. מה שהוא למעשה שואל כאן זה,

משתמש123:

יש פה מומחי Java שיכולים לבדוק את הבעיה שלי? מה שעלול לקרות, שגם אם למעשה זה לא קשור ל Java או מישהו שבכלל לא יודע Java יכול לענות על השאלה שלי.

יש מספר סיבות למה אנשים שכן יש להם את הידע, לא יודו בכך ויעזרו. בשאלה שלך אתה שואל יותר שאתה חושב שאתה שואל.

אתה מבקש מאנשים לקחת אחריות. אתה מטיל ספק בביטחון של אנשים ביכולות שלהם, ואתה חוסם אנשים אחרים ללא כל סיבה. לעתים קרובות אני עונה לאנשים על שאלות שקשורות לשפות-תכנות או ספריות, על אף שמעולם לא השתמשתי בשפות או בספריות האלו. מכיוון שהתשובה היא לא תלויה בשפה/ספריה אלא בחשיבה של מתכנת.

לחלופין, אפשר לראות זאת כ:

משתמש123:

יש לי שאלה ב Java אבל אני עצלן מדי בכדי לנסח אותה בצורה טובה אלא אם יש מישהו בצ'אט שכותב שהוא מוכן לענות.

שזה פשוט עצלנות. אם אתה לא מוכן לעשות את העבודה המועטה הזו ולהשקיע קצת בהודעה על מנת שאחרים יוכלו לעזור לך, למה שאחרים ירצו לעזור לך?

הפתרון הוא לא "לשאול בשביל לשאול", אלא פשוט לשאול. מישהו שרק מדי פעם נכנס לצ'אט לראות מה קורה, לא סביר שהוא יענה לשאלה שלך, אבל אם תתאר את הבעיה מראש הסיכויים שהוא יראה את השאלה וירצה לענות הרבה יותר גבוהים.

אז, לסיכום, אל תשאל "יש מומחי Java בסביבה?", אלא עדיף שתשאל "איך אני עושה את זה [בעיה] עם Java ו [עוד מידע שקשור השאלה]"
