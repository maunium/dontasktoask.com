Ζαχαρίας123:

Υπάρχει κάποιος εδώ ειδικευμένος στην Java που είναι πρόθυμος να δεσμευτεί στην εξετάσει του προβλήματος που αντιμετωπίζω, ό,τι κι αν είναι το πρόβλημα πραγματικά, είτε σχετίζεται με την Java ή όχι, ή αν κάποιος που δεν γνωρίζει τίποτα στην Java θα μπορούσε να μου ανταποκριθεί ασχέτως?

Υπάρχουν πολλοί λόγοι για τους οποίους κάποιοι που ΈΧΟΥΝ τη γνώση δεν θα ήθελαν να το παραδεχτούν. Θέτοντας την ερώτηση, στην πραγματικότητα ζητάς περισσότερα από αυτό που νομίζεις ότι ζητάς.

Τους ζητάς να αναλάβουν ευθύνη. Αμφησβιτάς την εμπιστοσύνη που έχουν αύτη στις ίδιες τις ικανότητές τους. Επίσης, απομακρύνεις τους άλλους χωρίς λόγο. Συχνά, απαντώ σε ερωτήματα σχετικά με γλώσσες προγραμματισμού ή βιβλιοθήκες λογισμικού που δεν έχω χρησιμοποιήσει ποτέ, διότι οι απαντήσεις είναι (πέρι τρόπο προγραμματιστή) κοινής λογικής.

Εναλλακτικά, μπορεί να ερμηνευθεί ως...

Ζαχαρίας123:

Έχω μια ερώτηση σχετικά με την Java, αλλά βαριέμαι πολύ να την συνθέσω σωστά, εκτός αν υπάρχει κάποιος στο κανάλι που μπορεί να την απαντήσει.

... που είναι απλά μια τεμπελιά. Εάν δεν είσαι διατεθειμένος να δουλέψεις στη λύσει μόνος σου, γιατί να το κάνουμε εμείς?

Η λύση δεν είναι να ζητήσεις εάν μπορείς να ρωτήσεις, αλλά απλά να ρωτάς. Κάποιος που είναι αδρανής στο κανάλι και κοιτάζει μόνο κάθε τόσο τι συμβαίνει δεν πρόκειται να απαντήσει σε ερωτήσεις «ζητώ να ρωτήσω», όμως εάν δώσεις μία ολόκληρη περιγραφή του προβλήματός σού, τότε υπάρχει περίπτωση να ενδιαφερθεί αυτός και να προσφέρει εθελοντικά μια απάντηση.

Συνοψίζοντας, λοιπόν, μην ρωτήσεις «Κανείς εδώ εξειδικευμένος στην Java?», αλλά αντίθετα ρώτησε «Πώς μπορώ να κάνω [Χ] με την Java και [άλλες σχετικές πληροφορίες]?»

Άλλα παρόμοια προβλήματα: Το πρόβλημα του Χ-Υ, Όχι Γεια. Περαιτέρω ανάγνωση: Πώς ζητάω μια καλή ερώτηση?, ή εάν έχεις περισσότερο χρόνο: Πώς Να Υποβάλλεις Ερωτήσεις Τον Έξυπνο Τρόπο.
