Ikke spør om å spørre, bare spør

Av og til på online chatterom jeg henger i, kommer noen inn og sier noe som ligner på:

Foobar123:

Er det noen Java eksperter her?

For flere grunner er dette dårlig etikett. Hva personen egentlig spør om er:

Foobar123:

Er det noen Java eksperter her som er villige til å sette av tid for å forstå problemet mitt, hva en det ender opp med å være, selv om det egentlig ikke er relatert til Java, eller hvis noen som egentlig ikke kan noenting om Java, svare på spørsmålet mitt?

Det er massevis av grunner for hvorfor folk som faktisk HAR kunnskapen, ikke vil innrømme det. Med å spørre, spør du om mer en det du egentlig tror du gjør.

Du spør folk om å ta ansvar. Du er spørrende mot folks sikkerhet i evnene deres. Du utestengenger også folk unødvendigvis. Ofte svarer jeg på spørsmål relatert til programmeringsspråk eller libraryer som jeg aldri har brukt, fordi svarene er (for programmerere) sunn fornuft.

Alternativt kan det bli sett på som..

Foobar123:

Jeg har ett spørsmål om Java men jeg er for giddaløs til å faktisk sette ord på det, hvis ikke det er noen som er villige til å svare.

..som er bare giddaløst. Hvis du ikke er villig til å gjøre arbeidet for å løse problemet ditt, hvorfor skal vi?

Løsningen er å ikke spør om å spørre, men å bare spør. Noen som henger rundt og bare av og til sjekker hva som skjer har lav sjanse for å svare på "spørre om å spør" spørsmålet ditt, men din faktiske problembeskrivelse kan vekke interessen deres og få dem til å svare.

Så, for å oppsummere, ikke spør "Er det noen Java eksperter her?", spør heller "Hvordan gjør jeg [problem] med Java og [annen relevant info]?"

Andre ligndende problemer: XY problemet, Ingen hallo. Mer lesing: Hvordan stiller jeg et bra spørsmål?, eller hvis du har mer tid: Hvordan stille spørsmål den smarte måten.
